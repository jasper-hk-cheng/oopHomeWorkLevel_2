    ## 程式架構與開發進程的制定 ##

    目前開發架構及順序如下:
    1. html 畫面規劃與開發。
    2. javascript 商業邏輯所需的物件建構子部分(依作業的設計模式主題實作各類別)。
    3. javascript user action 邏輯，應用 step 2 所設計的建構子來建立物件實作。
    4. 測試 step2, step3 的邏輯。
    5. javascript 動畫"關鍵畫面"(keyframes 或稱"關鍵幀")邏輯開發，公用程式固定以 animationUtils 為空間命名。
    6. 將動化邏輯嵌入商業邏輯，進行整合測試。
    7. 畫面最後調整，關鍵幀頭尾呈現方式調整。
    8. 製作plantUML圖
    制訂這些項目的最大目的，是希望未來可以實作更複雜的邏輯且配合更複雜的動畫，並達到易於維護的目的。

    未來目標:
    1. 加入自動化的測試程式，以提昇測試覆蓋率。
    2. 加入測試程式後，掩護動畫部分程式的重構。
    (因目前對動畫的技術了解有限，何者為較佳的重構架構尚無定案，故動畫js部分尚未遵循設計模式開發。)

*/
